# 技術型高中自然科學領域「探究與實作」教案產生器

## 這是什麼？

這是一個 OpenCode skill，能將任意教學主題、既有教案或上課內容，自動轉換成符合**台灣 108 課綱**的**技術型高中自然科學領域「探究與實作」教案**，並填入教育部技高自然科推動中心的**官方空白 Word 範本**。

支援科目：物理、化學、生物（A 版 / B 版）。

## 適用對象

- 技術型高中（高職）自然科教師
- 需要產出「探究與實作」正式教案的老師
- 想將既有上課內容改版為 108 課綱格式的老師

## 使用方式

在 ChatGPT／Claude 等支援 OpenCode skill 的平台中，只要提到以下關鍵字就會自動觸發：

- 「幫我寫一個探究與實作教案」
- 「把這份上課講義轉成 108 課綱格式」
- 「我需要技術型高中的自然科教案」
- 「幫我改版這份教案成探究與實作」

## 產出流程（9 步驟）

系統會依照以下 9 個步驟分析你的教學內容，並逐步產出對應的教案內容：

| 步驟 | 產出項目 | 說明 |
|------|----------|------|
| **1** | **三面九項** | 分析你的主題對應到 108 總綱核心素養的哪些項目（如 A2 系統思考、B1 溝通表達等），並說明理由 |
| **2** | **核心素養具體內涵** | 從課綱中取出每個選定項目的官方完整說明文字 |
| **3** | **學習內容** | 找出對應的課綱學習內容代碼（如 `PKa-V.2-8`），加上主題、次主題、主要概念與建議節數 |
| **4** | **議題融入** | 從 19 項議題（如科技教育、環境教育、資訊教育等）中選出 1-2 項最適合你的主題的，寫入融入說明 |
| **5** | **學習表現** | 將學習內容對應到學習表現三項目十一子項，並設計自訂檢核點 |
| **6** | **學習目標 + 學習脈絡** | 寫出 4-5 條具體可評量的學習目標，以及 4 階段的學習脈絡（發現問題→規劃研究→論證建模→表達分享） |
| **7** | **學習過程** | 整合主題名稱、教學時間、教材來源、教學資源、教學準備、教學說明、學習重點、情境規劃 |
| **8** | **教學歷程** | 將總節數拆成 4-8 個教學階段，每個階段寫明教學時間、教學資源、教學評量（對應檢核點） |
| **9** | **產生 Word 檔** | 將以上所有內容填入官方空白範本 docx，輸出正式教案 |

你可以選擇兩種模式：
- **逐步確認模式（預設）**：每做完一個步驟就停下來讓你看過、修改，再繼續下一步
- **一次產出模式**：一次跑完所有步驟，直接給你完整草案

## 可選補充教材

主要教案完成後，還可以額外要求以下補充教材（會直接附加在教案 docx 後面）：

1. **WSQ 學習單** — 給學生的填空式學習單（Watch 觀察 → Summarize 歸納 → Question 提問）
2. **學習表現檢核表** — 教師用的課堂觀察評分量表（精熟／良好／待加強三階）
3. **學習評量 + 評分規準** — 學生測驗卷（選擇 + 簡答）+ 教師詳解與給分標準

## 技術需求

系統在產出 docx 時需要使用以下工具（由 AI 自動呼叫，使用者無需自行安裝）：

- **Python 套件**：`python-docx`
- **PDF 搜尋**：`pdftotext`（poppler-utils）
- **PDF 轉換驗證**：`soffice`（LibreOffice）— 將 docx 轉 PDF 以視覺確認排版無誤

## ⚠️ 重要注意事項（給 AI 使用）

### 1. 範本有 BIG5 編碼問題

官方空白範本是在 BIG5 環境下製作的。用 python-docx 讀取後，所有中文字會變成亂碼（如 `'1.�椸�D�D�G'` 代替 `'1.單元主題：'`）。

**不要用中文文字去搜尋段落** — 永遠比對不到。改用**段落索引 + run 索引**的方式填入。

完整的索引對應表寫在 `references/template_structure.md` 的 Header 段落。

### 2. 合併儲存格的陷阱

python-docx 的 `table.cell(row, col)` 對於垂直合併的儲存格，會傳回同一個 XML 元素，導致不同行寫入的內容互相覆蓋。

**永遠使用 `scripts/docx_helpers.py` 的 `get_tr_list` / `get_tc` / `set_tc_text` 等函數**（以 raw XML 操作），不要直接呼叫 `table.cell()`。

特別需要注意有垂直合併的表格：
- **表格 1**（核心素養具體內涵）：col0 面向欄位有 vMerge
- **表格 6**（學習表現三項目十一子項）：col0 項目欄位有 vMerge
- **表格 9**（學習過程 + 教學歷程）：col0 有複雜合併

### 3. 表格 6 必須標示選取行

表格 6 是預先印好的靜態文字（學習表現三項目十一子項），**文字不能改**，但**必須把選取的子項行標上黃底**（FFE699），否則所有子項看起來都是未選取狀態。

使用 `highlight_table6.py` 的 `highlight_table6_in_doc()` 或 `collect_selected_from_table7()` 自動處理。

### 4. 填寫格式要求

- **學習內容代碼**：必須寫「編號 + 完整名稱」，如 `PKc-V.2-11 電流的磁效應`
- **學習表現**：同樣要寫「代碼 + 完整官方描述」
- **議題融入**：優先查 `assets/台灣_108_課綱19項議題融入.pdf`，而非課綱 PDF 的附錄

### 5. 一定要視覺驗證

產出 docx 後，必須用 LibreOffice 轉成 PDF，確認前 2-3 頁和最後一頁的表格邊框、合併儲存格、底色都正確無誤再交付。開發過程中確實發生過合併儲存格被覆寫的 bug，這個步驟救過很多次。

## 開發歷程

- 依據教育部技術型高中自然科學領域推動中心之官方教案範本 reverse-engineered
- 九步驟分析方法論源自課程設計者之原始 GEM.txt 流程
- 所有 Python 腳本均經過至少兩個真實教案（化學/3D列印、物理/電動馬達）的產出測試
